  Background/Aims/Objectives:

  • Our current leading idea for a capstone project is a Multi-Agent Board Game Intelligence. This will blend concepts from our choice of specialist certs Intelligent Sensing Systems and Architecting AI Systems.
  • The overall problem statement is to explore how we can create a multi-agent framework in the context of board games, where the AI model may be a foundation model, but we will utilize these in conjunction with frameworks and specialized skills for rule extraction, move prediction, and strategic decision making.
  • Develop a multi-agent framework where each functionality is handled by a specialized agent. Enable automatic rulebook comprehension using NLP and Retrieval-Augmented Generation (RAG).
  • Implement opponent modeling for next-move prediction using game tree search, probabilistic models, and sequence modeling.
  • Build opponent character profiling through feature engineering, clustering/classification, and reinforcement learning.
  • Ensure explainability by making AI decisions and player profiles interpretable.

  1. Zoom on Case Study: Azul is a turn-based, perfect-information, multiplayer, adversarial board game whose rules are deterministic yet whose difficulty from the combinatorics of drafting, placement constraints, and multi-player interaction (e.g., denial/blocking). Theofficial rules define a repeating Factory offer -> Wall-tiling -> Prepare next round loop, tile drafting from factories or the centre, pattern-line placement constraints, floor penalties, adjacency-based scoring, and end-of-game bonuses.

  1. One of the Deliverables: A Multi-Agent Decision Stack[c] 

that (a) remains correct via a deterministic Validator, (b) learns and adapts to unknown opponent "profiles" (imperfect styles), and © produces grounded, inspectable, user-facing explanations tied to search internals and rule-level semantics

  1. Validator (Authoritative[d]): deterministic rules + state transitions; the only component allowed to mutate game state
  2. Profiler: produces a brief distribution over opponent archetypes and/or a predictive opponent policy.
  3. Tactician[e]:chooses actions via MCTS/Minimax (optionally guided by learned priors.value) conditioned on Profiler output
  4. Explainer: renders a faithful explanation by consuming artefacts emitted by Validator/Tactician.
